## КРИТИЧЕСКОЕ

### C-1. Статус `'producing'` нарушает CHECK constraint
`worker.ts:1009` пишет `status='producing'`, но constraint в `001_youtube_module.sql:67-76` этот статус не включает, и ни одна миграция его не добавляла. Либо constraint был изменён руками в проде (см. C-2), либо каждый `handleProduce` падает на UPDATE.
**Fix:** миграция `ALTER TABLE yt_videos DROP CONSTRAINT ... ADD CONSTRAINT ... CHECK (status IN (..., 'producing'))` — и проверить фактический constraint в проде.

### C-2. Схема прода разошлась с репозиторием
- Таблица `nl_article_ideas` используется 4+ файлами (`app/api/ideas/*`, `lib/articles/resurface.ts`), но миграции для неё нет.
- Дыра в нумерации миграций 027–039 (14 номеров) — признак изменений напрямую через Supabase-консоль.
**Fix:** `supabase db diff --schema public` против прода, сгенерировать недостающие миграции. Без этого новое окружение/восстановление БД невозможно.

### C-3. Регрессия: `/api/auth/start` снова публичный
`middleware.ts:18` — путь остался в `PUBLIC_PATHS`, хотя комментарий утверждает обратное, а сам роут не вызывает `requireAuth()`. Это устаревший OAuth flow из прошлого аудита.
**Fix:** убрать из `PUBLIC_PATHS`; лучше — удалить старый flow `/api/auth/{start,callback}` целиком (активен `/api/youtube/oauth/*`).

## КАЧЕСТВО КОДА / АРХИТЕКТУРА

### HIGH

1. **worker.ts — монолит 1795 строк, 13 job-типов, 42 функции.** Уже есть `lib/worker/` с извлечёнными модулями — продолжить: handlers по одному файлу в `lib/worker/handlers/`, начиная с handleTranscribe / handleProduce / handleAnalyzeClips.
2. **Дубли helpers:** `worker.ts:227-245` (logJob/logChange/getVideo) дублируют `lib/process/helpers.ts` с расхождением поведения (worker глотает DB-ошибки, helpers кидают; STATUS_ORDER guard есть только в worker). Объединить в helpers, worker — импортировать.
3. **`getVideo()` = `select('*')`** (`worker.ts:243`) — тянет transcript (50–200KB) и producer_output (10–50KB) в каждом из 8 handlers, включая те, кому нужно 3 поля. Селективные select по handler'ам.
4. **Внешние fetch без таймаутов:** YouTube API в handlePublish (`worker.ts:918,947,964,969`), Unisender в handleNewsletterStats (`1392,1400`). `AbortSignal.timeout(30000)` — паттерн уже есть у Claude-вызовов (180s).
5. **`GET /api/tasks` без `.limit()`** — unbounded query.

### MEDIUM

- `components/layout/Sidebar.tsx:161` — всё ещё хардкод `CHANNELS` из `lib/channels.ts`: новый канал в БД не появится в сайдбаре без деплоя.
- `app/api/health/route.ts` — свой `createClient()` вместо `supabaseAdmin`.
- 33 `any` в worker.ts (claudeWithRetry, msg.content.filter, social_drafts.map).
- `playwright` (~130MB) в `dependencies`, используется только в carousel/export через dynamic import с fallback.
- Два lockfile: деплой идёт через npm → `bun.lock` лишний.

### LOW

- **Тестов нет вообще** — ни одного файла, ни test-скрипта при 116 роутах и worker'е. Приоритетные цели для первых unit-тестов: claudeWithRetry, STATUS_ORDER guard, парсинг таймкодов, SFTP-пути letters, идемпотентность enqueueProcessJob (источник бага #101).

## UX

### Что уже хорошо

Светлая тема при SSR починена (pre-paint script в layout.tsx); токен-система в globals.css; фокус-ринги; Sidebar с группировкой и accent-rail; calendar/admin/ideas/articles — образцовые состояния loading/error/empty; useUnsavedChanges + автосейв 30s в редакторах; StatusStepper; защита от двойного сабмита почти везде.

### HIGH

1. **Тихие провалы мутаций** — tasks, telegram, youtube list, carousels, clips: `handleSave` в `app/tasks/page.tsx:92-116` не проверяет `res.ok` — неудачное сохранение молча закрывает drawer и теряет ввод; `app/telegram/page.tsx:72-80` при ошибке не показывает ничего; `app/clips/page.tsx:21-29` рендерит фейковое «пусто» при падении API. **Fix:** сквозной свип `res.ok` + `toast.error`; ошибка не должна рендерить empty state.
2. **Два конкурирующих фидбек-механизма:** есть sonner + готовый `toastConfirm` в `lib/toast.ts`, но 15+ вызовов `alert()`/`confirm()` (youtube page — 8 шт, settings — 3, telegram, carousels, WhitePaper). Мигрировать.
3. **Зависшие job'ы без выхода для пользователя:** polling каждые 5s, но нет elapsed time, нет предупреждения по таймауту, нет кнопки retry при `error` (grep «Повторить» в UI — пусто). Учитывая историю бага #101 — добавить elapsed + retry.
4. **Мобильной версии нет:** фиксированный сайдбар 180px, hover-flyouts (не работают на touch), ~28 файлов с responsive-классами. Off-canvas drawer ниже `md` — отдельная задача.

### MEDIUM

- **Дизайн-волна остановилась на ~30%:** `ui/input.tsx`/`textarea.tsx` импортированы 1 раз; 25 файлов с raw `<input>`; Button в 11 файлах, остальные — самодельные кнопки без focus/press-состояний. Худшие отстающие: youtube list, carousels, settings, login.
- Toaster захардкожен `theme="dark"` (`layout.tsx:44`) — тёмные тосты в светлой теме; `.shimmer` невидим в light.
- Реальные light-theme breakers: `ThumbnailGallery.tsx:59,68`, `TranscriptViewer.tsx:82,89`, `SidebarFlyout.tsx:29`, градиент в `app/youtube/[id]/page.tsx:323`.
- `app/settings/page.tsx` — 1124 строки, 30 useState; articles/[id] — 900+. Нарушают собственный лимит 800.
- A11y: 12 aria-атрибутов на всё приложение; модалки без `role="dialog"`/focus trap; кликабельные карточки-div без клавиатуры; hover-only flyouts недоступны с клавиатуры.

### LOW

- Языковой микс минимален (UI стабильно русский, англ. в техлейблах) — не трогать.
- Пустые колонки TaskBoard без CTA; `EmptyState` используется только в 3 файлах.
