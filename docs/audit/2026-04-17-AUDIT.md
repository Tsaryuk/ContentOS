# ContentOS — аудит кода, безопасности и план миграции auth

**Дата:** 2026-04-17
**Ветка:** `claude/wonderful-moser`

## Критические проблемы (блокеры)

### Безопасность
1. **middleware.ts** — проверяет только наличие cookie, не расшифровывает. Любой мусорный cookie проходит как валидная сессия.
2. **Routes без `requireAuth()`**: `/api/youtube/sync` (mass delete), `/api/transcript`, `/api/process/{transcribe,generate,publish,produce,thumbnail}`, `/api/thumbnail/{upload,generate,save-assets}`, `/api/projects` GET/POST, `/api/auth/switch`.
3. **OAuth CSRF** — оба flow (`/api/auth/*` и `/api/youtube/oauth/*`) без `state` параметра.
4. **Path traversal**: `lib/articles/publish.ts:81` — `blog_slug` интерполируется в `path.join` без санитайза.
5. **SSRF**: `app/api/articles/cover/route.ts:85` — `fetch(body.fal_url)`; `app/api/thumbnail/generate/route.ts:263` — `body.photos[]` и `referenceUrl` как `image_urls` к fal.ai без whitelist.
6. **Mass assignment**: `PATCH /api/articles/[id]`, `PATCH /api/newsletter/issues/[id]`, `PATCH /api/clips/candidates` — `supabaseAdmin.from(...).update(body)` без field-allowlist.
7. **execSync с user data**: `worker.ts:412,436` — `yt_video_id` интерполируется в shell-команду yt-dlp/ffmpeg.
8. **RLS фактически отключен** — все политики в миграциях `USING (true)`.
9. **Конфликт миграций**: два файла с префиксом `012_` (`012_carousel_voices.sql`, `012_users.sql`).
10. **CHECK constraint** для `yt_videos.status` не включает `'producing'`, но worker использует этот статус.

### Высокий приоритет
- IDOR на DELETE: `/api/accounts/[id]`, `/api/channels/[id]`, `/api/telegram/posts/[id]` — нет ownership check.
- XSS через `dangerouslySetInnerHTML`: `app/articles/[id]/page.tsx:605`, `components/telegram/PostEditor.tsx:204`, `components/newsletter/EditorPanel.tsx:368`, `components/newsletter/ArticlePanel.tsx:178` — HTML из БД рендерится без sanitize.
- **Secrets в plaintext**: `google_accounts.refresh_token`, `yt_channels.refresh_token`, `tg_accounts.session_string`.
- **Нет rate limiting**: `/api/auth/login`, `/api/newsletter/subscribe`, `/api/telegram/auth/start`, AI-генерация.
- **N+1**: `worker.ts` handleProduce (3 запроса social_drafts), `app/api/youtube/sync` (300 upserts в batch ОК, но delete по одному).
- **Отсутствующие индексы**: `yt_videos(published_at)`, `yt_videos(channel_id, published_at)`, `yt_channels(is_active)`, `yt_social_drafts(video_id, platform)` (UNIQUE).
- **Дублирование createClient()**: `app/api/thumbnail/{generate,upload,save-assets}` и `worker.ts` создают свои клиенты вместо `lib/supabase.ts`.
- **Дубли функций**: `worker.ts` повторяет `updateVideoStatus/logJob/logChange/getVideoWithChannel` из `lib/process/helpers.ts`.
- **Два OAuth flow**: `/api/auth/{start,callback}` устарел, активен `/api/youtube/oauth/*`.

### Средний приоритет
- `error.message` из Supabase leak в HTTP 500 ответах (enumeration схемы БД).
- `ecosystem.config.js` загружает `.env.local` в PM2 env → доступно через `pm2 show`.
- 25+ `any` в `worker.ts`.
- SSR светлой темы сломан: `app/layout.tsx:16` — жёсткий `<html className="dark">`.
- Мёртвый код: `lib/process/thumbnail-generator.ts`, `components/layout/ProjectChannelSwitcher.tsx` (проверить использование).
- `lib/channels.ts` — хардкод каналов, Dashboard не использует БД.

### Низкий приоритет
- `app/api/auth/logout`: удаляет cookie `contentos_auth` (неправильное имя).
- Login timing leak (bcrypt compare только для существующих).
- Нет security headers (`CSP`, `X-Frame-Options`, `Referrer-Policy`).
- Два lockfile (`bun.lock` + `package-lock.json`).
- `playwright` в `dependencies` вместо `devDependencies`.

## Матрица API routes × auth

Полный список с состоянием auth — в отдельных разделах аудита. После PR 1 каждый роут должен иметь `requireAuth()` или `requireAdmin()` первой строкой. Публичные остаются: `/api/auth/login`, `/api/auth/logout`, `/api/auth/callback`, `/api/auth/session` (возвращает минимум), `/api/youtube/oauth/{start,callback}`, `/api/newsletter/subscribe`, `/api/health`.

## План PR

- **PR 1** — Security critical: middleware decrypt, requireAuth везде, OAuth state, blog_slug sanitize, SSRF whitelist, mass-assignment allowlist, execFile вместо execSync.
- **PR 2** — Security high: IDOR ownership, sanitize HTML, rate limiting, security headers, encrypt refresh_token и session_string at rest.
- **PR 3** — Perf + code quality: N+1 fixes, миграция с индексами, dedupe createClient, worker → helpers, fix миграции 012, fix 'producing' CHECK, удалить мёртвый код, удалить старый OAuth flow, fix SSR theme.
- **PR 4** — Миграция на Supabase Auth (`@supabase/ssr`), `public.profiles`, RLS политики, сброс паролей существующим юзерам, удаление iron-session.

## Решения по Supabase Auth (принято автономно)

- UI state (`activeProjectId`, `activeChannelId`) → localStorage + unsigned cookie для SSR.
- Миграция пользователей → сброс паролей, создание `auth.users` с теми же UUID (FK не ломаются).
- Роль `admin` → `public.profiles.role` + дублировать в `auth.users.app_metadata.role` для middleware-доступа без запроса.
- `/api/auth/{start,callback}` старый flow → удалить, merge в `/api/youtube/oauth/*`.
- Self-signup → отключён (admin-only create).
- MFA — не внедряем. Password recovery — включить.
