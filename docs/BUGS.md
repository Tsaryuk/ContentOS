## Open

### B-01: Таймаут на "generating" зависает без recovery
- **Где:** https://huuman.ru/youtube/5bc3d949-9b9e-4c13-80bd-f865d6b72f2f
- **Что происходит:** после 15+ минут в статусе `generating` видео падает в `error`, хотя фактически работа выполнена (результаты сохранены).
- **Ожидалось:** либо статус корректно обновляется на завершение, либо падение в error только когда работа реально не выполнена.
- **Приоритет:** high (блокирует production flow)
- **Статус:** open — диагностика после Sentry+pino (Sprint 1.1/1.2)
- **Гипотеза:** heartbeat/timeout в worker обрывает job после N минут, не проверяя финальное состояние БД. Нужны логи чтобы увидеть timeline.

### B-02: Обложки "гость + я" требуют ручной refresh страницы
- **Где:** Thumbnail Studio (`/youtube/[id]` → раздел генерации обложек)
- **Что происходит:** после генерации с шаблоном `duo` (гость+я) UI пишет ошибку `load file`, нужно F5 чтобы увидеть готовые обложки. С шаблоном `solo` (только гость) — обновляются сразу.
- **Ожидалось:** оба шаблона ведут себя одинаково — обложки появляются без перезагрузки.
- **Приоритет:** medium
- **Статус:** open
- **Гипотеза:** race condition в обновлении `producer_output.thumbnail_urls_by_template` для шаблона duo, либо polling не ловит изменение.

### B-03: Custom reference — непредсказуемый результат
- **Где:** Thumbnail Studio → шаблон `custom` (свой референс)
- **Что происходит:** непонятно какой промпт передаётся в fal.ai и передаётся ли реф. Итог неконтролируемый.
- **Ожидалось:** видимый (или логируемый) финальный промпт + гарантия что custom-реф попадает в `image_urls`.
- **Приоритет:** medium
- **Статус:** open
- **Исправление:** показать финальный промпт в UI перед генерацией + проверить путь `referenceUrl → safeReference → image_urls` после PR 1 (SSRF whitelist мог отбросить реф если он не из allow-list).

### B-04: Для видео (не подкастов) нужен другой thumbnail reference
- **Где:** Thumbnail Studio
- **Что происходит:** сейчас один thumbnail-style-prompt на канал (`channel.rules.thumbnail_style_prompt`). Для видео до 50 минут нужен другой реф/промпт чем для длинных подкастов.
- **Ожидалось:** различать типы контента (podcast / shorts-style video), каждый со своим референсом и стилевым промптом.
- **Приоритет:** medium
- **Статус:** open
- **Исправление:** добавить `content_type` → разные preset'ы обложек; либо per-video-override в UI.

### B-05: Белый лист (article editor) — светлая тема ломается + нужен fullscreen
- **Где:** https://huuman.ru/articles/915b9274-5201-4d23-80b6-5245fd69ef53, `WhitePaper` компонент
- **Что происходит:** (a) в light theme стили не грузятся. (b) белый лист не занимает весь экран монитора — надо сделать fullscreen выше окна браузера.
- **Ожидалось:** (a) стили работают в обеих темах. (b) fullscreen на весь монитор (Fullscreen API / F11-style).
- **Приоритет:** medium
- **Статус:** open
- **Исправление:** (a) заменить hardcoded цвета на токены в `components/articles/WhitePaper.tsx`. (b) кнопка "Fullscreen" → `document.documentElement.requestFullscreen()`.

### B-06: "Черновик независим от редактора" — отвлекающий hint, удалить
- **Где:** редактор статей, подсказка "Черновик независим от редактора. В редактор — только по твоей команде."
- **Что происходит:** текст отвлекает пользователя и не несёт ценности.
- **Ожидалось:** убрать.
- **Приоритет:** low
- **Статус:** in-progress
- **Исправление:** grep + удалить.

### B-07: AI-стилист падает с "Load failed" на длинных текстах
- **Где:** https://huuman.ru/articles/* → Белый лист → кнопка "AI-стилист", `/api/articles/style-edit`
- **Что происходит:** на длинных черновиках Safari ~через 60с отменяет fetch (nginx видит 499), в UI alert "Ошибка: Load failed" (пользователь читает как "load file"), текст не меняется.
- **Ожидалось:** редактура доезжает до конца без таймаута и появляется в текстарея.
- **Приоритет:** high
- **Статус:** in-progress — стриминг ответа Anthropic на клиент (keepalive + прогрессивный апдейт textarea), заголовок `X-Accel-Buffering: no` чтобы nginx не буферил.
